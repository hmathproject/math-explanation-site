# PDF_QUALITY_CHECK.md — サイト向けPDF品質チェック標準

このファイルは `site/docs/PDF_STANDARD.md` で定めた integrated_exp フォーマットで制作した PDF の
**ビルド後品質確認手順**を定めた運用文書です。
PDF を作るたびに、ここに書かれた順でチェックしてからリリースしてください。

---

## 1. 想定される品質問題の分類

### A. 機械的に検出できる問題（ビルド直後・PDF を開く前）

| コード | 問題 | 検出方法 |
|---|---|---|
| A1 | xelatex ビルド失敗 | ログに `Output written on` が出ているか |
| A2 | ページ数超過（3ページ超） | ログの `(N pages)` を確認 |
| A3 | PDFサイズ異常小（図抜け疑い） | `ls -lh` でサイズ確認（図あり：70KB 未満は要注意） |
| A4 | figures/ に参照 PNG が存在しない | ビルド前に `ls figures/` で確認 |
| A5 | COL_LEFT_BEGIN / COL_MID / COL_END の対応漏れ | 原稿を `grep COL` で確認（同数でなければエラー） |

### B. 目視が必要な問題（PDF を開いて確認）

| コード | 問題 | 発生しやすい場面 |
|---|---|---|
| B1 | 図中の文字衝突（y 軸ラベル vs ケースタイトル） | ケースタイトルが長いとき |
| B2 | 数値ラベルが放物線・曲線と重なる | y 値が 0 に近い点ラベル |
| B3 | 初出記号（a, D 等）の定義がない | 判別式・二次係数を突然使用したとき |
| B4 | 2段組崩れ（右列幅超過・左右不均衡） | 長い数式を右列に置いたとき |
| B5 | raw LaTeX 断片が本文に漏れる | `\textbackslash\textbf` など |
| B6 | 太字が正しく太字で表示されない | `build_pdf.py`（旧）を誤使用したとき |
| B7 | 数式の崩れ（分数・cases 環境） | `\\[4pt]` の誤使用 |
| B8 | カバーの収録問題リストと実際の4問が不一致 | 単元変更後にカバー更新を忘れたとき |
| B9 | 教材として不親切な表現（論理の飛躍・答案語に説明が混入） | サイト記事をそのまま流用したとき |

## 2. 機械的チェック（ビルド後・PDF を開く前に確認）

```
[ ] xelatex がエラーなく完了している
    → build_exp_pdf.py の出力に "Output written on ... (N pages)." があるか
[ ] 個別 PDF のページ数 ≤ 2（カバーなし）
    → ログの "(N pages)" を確認。3以上なら LAYOUT_RULES.md の圧縮手順を実施
[ ] 個別 PDF のサイズ ≥ 70KB（図あり前提）、パック ≥ 300KB
    → ls -lh で確認。サイズが極端に小さければ図の欠落を疑う
[ ] figures/ に <!-- FIGURE: *.png --> で参照した PNG が存在する
    → ビルド前に ls figures/ で確認。なければ site/assets/images/ からコピー
[ ] 原稿内の COL_LEFT_BEGIN / COL_MID / COL_END が同数対応している
    → grep "COL_LEFT_BEGIN\|COL_MID\|COL_END" 原稿.md で行数を確認
```

---

## 3. 目視チェック（PDF を開いて実際に確認）

```
[ ] raw LaTeX 断片が本文に露出していない
    → \textbackslash\textbf、\begin{...} が本文に見えたら build_exp_pdf.py を使っているか確認
[ ] **太字** が正しく太字で表示されている
    → 細い文字に見えたら build_pdf.py（旧）を誤使用している可能性
[ ] 図が ## 方針 セクション内に正しく表示されている
    → 「図が見つかりません」表示、空白、図の欠落がないか
[ ] 図の y 軸ラベル "$y$" とケースタイトルが重なっていない
    → 近接する場合は図生成スクリプトの Y_TITLE_OFFSET を大きくする（後述）
[ ] 図の数値ラベル（a, D=0 等）が放物線・曲線と重なっていない
    → 重なりがある場合は白 bbox + zorder=7 が付いているか確認
[ ] 初出記号（a, D など）に定義または説明がある
    → 「a > 0 より」の前に \( a \) が何を指すか文脈で分かるか確認（後述の導入ルール参照）
[ ] 2段組が左右バランスよく表示されている（右列の幅超過がない）
    → 右列の文字が左列にはみ出している場合は右列のテキストを短くする
[ ] 数式が正しく表示されている（分数・cases・不等号）
    → 特に cases 環境と dfrac の表示を確認
[ ] カバーの収録問題リストが実際の4問の内容・順序と一致している
[ ] 単元トップ・各記事の PDF ダウンロードリンクが /assets/pdf/... の正しい URL
[ ] グラフ端に水平な黒線（clipping 痕）が出ていない
    → 放物線の端が不自然に水平になっていたら np.clip() が y データに適用されている
    → 修正: np.clip() を削除し curve_ylim() で ylim を再計算してスクリプト再実行
```

## 4. 図レイアウト監視ルール

### 現状スクリプトの実測値（gen_figures*.py 共通）

| 要素 | 現在の値 | 推奨値 | 備考 |
|---|---|---|---|
| y 軸ラベル "$y$" の y 位置 | `Y_HI + 0.27` | 現状維持 | 高さは十分 |
| ケースタイトルの y 位置 | `Y_HI + 0.12` | `Y_HI + 0.20` | ← 衝突リスクあり。0.20 に引き上げ推奨 |
| ケースタイトルの fontsize | `11` | `10` | ← 1pt 縮小で衝突リスク低減 |
| tight_layout の pad | `0.2` | `0.5` | サブプロット間の余白確保 |
| 数値ラベルの bbox | 白 bbox + zorder=7 | 現状維持（必須） | 放物線との重なり防止 |

**衝突リスクの根拠**: y 軸ラベル（Y_HI + 0.27）とケースタイトル（Y_HI + 0.12）の垂直ギャップは **0.15 単位**。
fontsize 10〜11 の文字高さに対してギリギリで、タイトルが長い場合に視覚的近接が生じやすい。

### 新スクリプト作成時のテンプレートルール

```python
# ── ラベル位置定数（冒頭で必ず定義する）──────────────────
Y_TITLE_OFFSET      = 0.20   # ケースタイトルの Y_HI からのオフセット
Y_YLABEL_OFFSET     = 0.27   # y 軸ラベルのオフセット（現状維持）
CASE_TITLE_FONTSIZE = 10     # ケースタイトルのフォントサイズ
TIGHT_LAYOUT_PAD    = 0.5    # tight_layout の pad

# ── 必須の白 bbox（全数値ラベルに適用）──────────────────
_lbl_bbox = dict(boxstyle="square,pad=0.05", facecolor="white", edgecolor="none")

# ── 配置ルール ────────────────────────────────────────────
# ケースタイトル : ax.text(cx, Y_HI + Y_TITLE_OFFSET,  title, fontsize=CASE_TITLE_FONTSIZE, ...)
# y 軸ラベル    : ax.text(0.04, Y_HI + Y_YLABEL_OFFSET, r"$y$", ...)
# 数値ラベル    : ax.text(xv, lbl_y, lbl, ..., bbox=_lbl_bbox, zorder=7)
```

### スクリプトで防げること vs 目視が必要なこと

| 対応 | 内容 |
|---|---|
| スクリプトで対応可 | Y_TITLE_OFFSET・CASE_TITLE_FONTSIZE の定数管理、白 bbox の全適用 |
| 目視が必要 | ケースタイトルが長い文字列（複雑な条件式）の折り返し、点ラベルの y 値が近い場合の重なり |

## 5. 初出記号の導入ルール

### 対象となる記号と導入タイミング

| 記号 | 意味 | 導入タイミング | 記述例 |
|---|---|---|---|
| \( a \) | 二次係数（最高次の係数） | 解法の流れ または 方針の最初の言及時 | `\( a = 1 > 0 \) より放物線は上に凸` |
| \( D \) | 判別式 \( D = b^2 - 4ac \) | 方針 または 解法の流れの冒頭 | `判別式 \( D = b^2 - 4ac \) の符号が〜` |
| \( \alpha, \beta \) | 方程式の根 | 根を「おく」直前 | `方程式の根（解）を \( \alpha, \beta \)（\( \alpha < \beta \)）とおく` |
| \( f(x) \) | 二次関数の式 | 問題文で定義済みなら不要。途中導入なら定義する | — |

### 導入ルール（単元内で最初の1回のみ定義）

```
良い例（解法の流れに組み込む）:
  - \( a = 1 > 0 \) より放物線は上に凸

良い例（方針に組み込む）:
  判別式 \( D = b^2 - 4ac \) の符号が放物線とx軸の交わり方を決める。

悪い例（突然使う）:
  D > 0 だから根が2つある    ← D の説明なし

悪い例（冗長な定義段落を設ける）:
  ここで a は ax²+bx+c の ax² の係数を表す定数である    ← 長すぎ
```

**定義の長さの基準**: 数式1本 + 日本語1句で十分。定義のために段落を設けない。

**例外**:
- 問題文自体に「\( a > 0 \) とする」と明示されている → 解答内で再定義不要
- 高校生が暗記済みの記号（\( x \), \( y \), \( \pi \) 等）→ 定義不要
