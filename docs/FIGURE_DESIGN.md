# FIGURE_DESIGN.md — 図設計自立判定ルール

このファイルは「テーマ骨格を作るとき、Claude Code が図の必要性を自立的に判定するための標準ルール」を定めた運用文書です。
チャットが変わった場合はまずこのファイルを読んでください。

---

## 1. 判定フローチャート（テーマ設計の最初に実行）

テーマ骨格を作る前に、以下の Step を順に確認する。

```
Step 1: Q5 に当たるか？
        テーマ名・記事タイトルに「グラフ・図形・軌跡・領域・面積・交点・移動」が含まれる
        → YES → 図必須 に確定

Step 2: Q4 に当たるか？
        テーマの核心が「どこに交点があるか」「どの範囲にあるか」という位置・範囲の問い
        → YES → 図必須 に確定

Step 3: Q1 + Q3 を同時に満たすか？
        Q1: 「なぜその条件か」を説明するとき、グラフや図がないと伝わらない
        Q3: 図がないと条件・結論が「覚えるもの（暗記）」に見えてしまう
        → BOTH YES → 図必須

Step 4: Q2 だけ当たるか？
        場合分けの根拠が「x軸・軸・頂点・区間・領域の位置関係」
        → YES → 図推奨（Q1・Q3 も合わせて総合判断）

Step 5: Q1〜Q5 いずれも当たらない → 図任意 or 不要
```

---

## 2. 5つの判定基準（Q1〜Q5）

| 記号 | 確認内容 |
|---|---|
| Q1 | 「なぜその条件か」を説明するとき、グラフや図が必要か |
| Q2 | 場合分けの根拠が「x軸・軸・頂点・区間・領域の位置関係」か |
| Q3 | 図がないと条件・結論が「覚えるもの（暗記）」に見えてしまうか |
| Q4 | テーマの核心が「どこに交点があるか」「どの範囲にあるか」か |
| Q5 | テーマ名・記事タイトルに「グラフ・図形・軌跡・領域・面積・交点・移動」が含まれるか |

---

## 3. 4段階の判定レベルと骨格への影響

| レベル | 定義 | 骨格段階での対応 |
|---|---|---|
| **図必須** | 図なしで「なぜその条件か」が伝わらない | 各記事の図設計メモ（§4-A）・スクリプト命名（§4-B）・placeholder（§4-C）を出す |
| **図推奨** | 図があると理解が深まるが、代数説明でも通じる | 骨格段階で図の方針メモを1行出す（任意） |
| **図任意** | 補助的に使える図があるが、テーマの本質ではない | 言及不要 |
| **図不要** | 計算・論理が主体で図は不要 | 言及不要 |

## 4. 図必須テーマで骨格段階に出す内容

### A. 各記事の図設計メモ（1記事につき1メモ）

```
記事名: [例] 解の配置（2解が同符号）
図の役割: 2解がともに正/ともに負の場合に、放物線がx軸に対してどこで交わるかを横並び比較する
ケース数: 2（ともに正 / ともに負）
figsize案: (9.0, 4.5)（2ケース標準）
スクリプト案: scripts/gen_figures_rl1.py
出力PNG案: assets/images/quadratic-root-location-same-sign-combined.png
図の挿入位置: 「なぜその条件か」セクションの直前（条件の根拠を視覚で先に示す）
```

### B. スクリプトファイル名の命名規則

形式: `gen_figures_[テーマ略称][記事番号].py`

| テーマ | 略称 | 例 |
|---|---|---|
| 解の配置 | `rl` | `gen_figures_rl1.py`（記事1）〜 `gen_figures_rl4.py` |
| 積分（数II）面積 | `in` | `gen_figures_in1.py`〜 |
| 漸化式 | `rr` | `gen_figures_rr1.py`〜 |
| 平行移動・対称移動 | `tr` | `gen_figures_tr1.py`〜 |
| 三角関数グラフ | `tg` | `gen_figures_tg1.py`〜 |

既存スクリプト命名（参考）:
- `gen_figures_q3.py` / `gen_figures_q4.py` … 最大値・最小値（最小3ケース / 最大2ケース）
- `gen_figures_qi1.py` 〜 `gen_figures_qi4.py` … 二次不等式（4記事）

### C. 記事テンプレートの図 placeholder

骨格段階では以下の形式で placeholder を入れる（スクリプトは後工程で作成）:

```markdown
## なぜその条件か

（説明文）

![（図の説明文）](/assets/images/（ファイル名）-combined.png)

（各ケースの詳細説明）
```

placeholder の前後の見出し・文章構成も骨格段階で決めておく。
「図があるべき位置」を骨格で確保することが目的。

## 6. テーマ別の判定結果一覧

| テーマ | 図必要度 | 主な根拠 |
|---|---|---|
| 最大値・最小値 | **必須** | 軸の位置で最小点が変わる → Q1・Q2・Q3 すべて該当 |
| 二次不等式 | **必須** | 解 = 放物線がx軸より上の範囲 → Q1・Q4 直接該当 |
| **解の配置** | **必須** | 解の位置条件を放物線の形で読む → Q1・Q2・Q3 すべて該当 |
| グラフと直線（交点・共有点） | **必須** | 交点の個数・位置がテーマの核心 → Q4・Q5 |
| 積分（数II）面積計算 | **必須** | 積分 = 面積 → 図なしでは概念が伝わらない → Q1・Q3・Q5 |
| 軌跡 | **必須** | 動点が描く図形 = 軌跡そのもの → Q4・Q5 |
| 領域と線形計画法 | **必須** | 不等式の解 = グラフの領域 → Q1・Q4・Q5 |
| 三角関数グラフと周期性 | **必須** | 周期・振幅・グラフの変形がテーマの核心 → Q4・Q5 |
| ベクトルと図形 | **必須** | 空間的推論 = 図が言語 → Q1・Q3 |
| 平行移動・対称移動 | **必須** | 移動後のグラフ形状 = 視覚で確認 → Q1・Q2・Q5 |
| 正弦定理・余弦定理 | **必須** | 三角形の辺・角・外接円 → Q1・Q5 |
| 式の決定 | **推奨** | 条件を代入して連立 → 代数で可能だが図で文脈が出る |
| 漸化式 | **推奨** | 数列の収束・振動を図で視覚化できるが代数で完結可能 |
| 場合の数・確率 | **任意** | 樹形図・ベン図が補助として使える |
| 数学的帰納法 | **任意** | 論理構造が主体 → 図は補助 |
| Σ計算 | **不要** | 純粋な計算テーマ |
| 不定積分の計算 | **不要** | 計算手法テーマ（面積応用は別枠） |

```
    → 伝わらない → 図必須 の再判定が必要
[ ] 「場合分けの根拠」「なぜその条件か」セクションに図が関連付けられているか
```

### 「暗記テーマに見えていないか」追加確認

以下の状態なら図が足りていない可能性が高い:

- 「条件1: D≥0」「条件2: f(0)>0」... と列挙するだけで「なぜか」がない
- 「まとめ」セクションに条件だけあって図が引用されていない
- 図は「あると便利」ではなく「なぜを説明する唯一の手段」として扱われているか

## 8. グラフ clipping 防止ルール（必須）

### 問題の概要

`np.clip()` を曲線の y データに適用してから `ax.plot()` に渡すと、クリップ境界で y 値が上限/下限に張り付く。
matplotlib はその「水平に並んだ点列」をそのまま折れ線として描くため、グラフ端に**不自然な水平黒線**が出現する。

```
悪い例（絶対に使わない）:
    y = np.clip(2*x**2 - 5*x + 3, Y_LO, Y_HI)   # ← y データに clip → 水平線が出る
    ax.plot(x, y, ...)

良い例（この方法を使う）:
    y = 2*x**2 - 5*x + 3                          # clip しない
    y_lo, y_hi = curve_ylim(y, extra_y=[...])      # ylim を実データから計算
    ax.set_ylim(y_lo - 余白, y_hi + 余白)          # matplotlib に境界処理を任せる
    ax.plot(x, y, ...)
```

### curve_ylim() ヘルパー関数（全スクリプト共通）

```python
def curve_ylim(y_arr, extra_y=(), margin=0.15):
    """
    曲線データと注釈 y 座標から安全な ylim を計算する。

    np.clip() の代わりにこの関数で ylim を決め、ax.set_ylim() に渡す。
    - y_arr  : 曲線の y 値配列（np.ndarray）
    - extra_y: マーカー・ラベル等の追加 y 値リスト
    - margin : y スパンに対する上下余白の割合（デフォルト 15%）
    Returns : (y_lo, y_hi)
    """
    y_all = list(y_arr) + list(extra_y)
    y_min, y_max = min(y_all), max(y_all)
    span = max(y_max - y_min, 1.0)
    return y_min - margin * span, y_max + margin * span
```

### スクリプト作成・修正のチェックリスト

```
[ ] y データに np.clip() を適用していないか
    → ax.plot(x, np.clip(y, ...), ...) の形が1行でもあれば即修正
[ ] ylim を curve_ylim() または実データ範囲から計算しているか
    → Y_LO / Y_HI をハードコードしたまま使っていないか
[ ] x 範囲の端点で y 値が ylim を大幅に超えていないか
    → 超えている場合は x 範囲を絞るか curve_ylim() に extra_y を追加する
[ ] 目視確認: グラフ端に水平線・台形状の切り欠けが出ていないか
```

### 発見時の対処フロー

1. 水平線が見えたら、y データに `np.clip()` が使われていないか確認する
2. `np.clip()` を削除し、`curve_ylim()` ヘルパーで ylim を再計算する
3. x 範囲の端点での y 値を計算し、ylim 内に収まることを確認する
4. スクリプトを再実行して PNG を再生成する
5. PDF も再ビルドする（図が PNG として埋め込まれるため）
