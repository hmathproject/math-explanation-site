## 1. テーマ体系の設計原則

**基本単位: 1テーマ = 単元トップ + 4記事前後 + PDFパック**

| 原則 | 内容 |
|---|---|
| 典型つまずき単位で切る | 教科書の節名ではなく「このタイプ苦手」と感じる問題パターンで切る |
| 図説で「なぜ」が説明できること | 場合分けが図で見える、解の意味がグラフに対応する |
| case構造を持つこと | 2〜4ケースの横並び比較記事が作れる（既存パイプラインの強み） |
| 検索クエリと対応すること | 「解の配置 条件」「漸化式 解き方」のように検索される切り口 |

### 粒度の基準

- **適正**: 4記事±2、1PDFパックにまとまる、ケース分類が明確
- **広すぎ → 分割**: 例「三角関数 全般」→「グラフと周期性」「加法定理と合成」「方程式・不等式」に分割
- **狭すぎ → 統合**: 例「頂点の符号判定」単独 →「式の決定」テーマに統合

### 表現ルール

- **「根」は使わない。方程式の解は必ず「解」と表記する。**
  - 例: 「2解がともに正」「実数解が存在する」

## 4. 粒度調整方針

### 分割が必要なケース

| 元テーマ（広すぎ） | 分割後 |
|---|---|
| 三角関数（全般） | グラフと周期 / 加法定理と合成 / 方程式・不等式（3テーマ） |
| 積分（数II全般） | 基本計算 / 面積計算（2テーマ） |
| 図形と方程式（全般） | 直線・円 / 軌跡 / 領域（3テーマ） |
| 数列（全般） | 等差等比 / 漸化式 / Σ・帰納法（3テーマ） |

### 統合が必要なケース

| 元テーマ（狭すぎ） | 統合後 |
|---|---|
| 「頂点の符号判定」単独 | →「式の決定」に吸収 |
| 「条件付き係数決定」 | →「解の配置」または「式の決定」に吸収 |
| 「共有点の個数判定」単独 | →「グラフと直線」2記事の1記事として吸収 |

### 重複回避

- 二次関数「グラフと直線の面積」と 数II積分「面積計算」は別テーマとして立てる
  → 前者は図形的観点、後者は積分計算手法の観点で切り口が違う
- 三角比（数I）と三角関数（数II）は別テーマ
  → 前者はラジアン不使用、後者は単位円・周期が中心

## 6. 新テーマをロードマップに追加するときの手順

1. **テーマ名を決める**: 学習者目線の典型問題名（教科書節名は避ける）
2. **粒度チェック**: 4記事±2に収まるか確認。広ければ分割、狭ければ統合
3. **このファイルの §2 テーブルに追記**: 状態は「将来」で追加
4. **制作着手時**: 状態を「🔜」に更新し、§3 に詳細案を追記
5. **完成時**: 状態を「✅」に更新
